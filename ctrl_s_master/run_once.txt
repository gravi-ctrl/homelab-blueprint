The "Fresh Install" Runbook

Run this only once when setting up the server for the first time or migrating to a new OS.


1. System Prep & Dependencies
# Copy your project folder to ~/scripts/ctrl_s_master first.
cd ~/scripts/ctrl_s_master
# Make scripts executable
chmod +x setup.sh run.sh update.sh tests.sh src/master_automation.py src/_tools/*.py
# Run the one-click installer
./setup.sh

---

2. The Secret Key Setup
# Create the auto-unlock key
# Replace 'YOUR_REAL_PASSWORD' with the actual container password
sudo sh -c 'echo "YOUR_REAL_PASSWORD" > /root/.vc_secret'
sudo chmod 600 /root/.vc_secret

---  

3. The Container Initialization (The "Skeleton")
This creates the folders AND moves your local .env into the secure container.
# 1. Create Mount Point
sudo mkdir -p /mnt/secure_vaults

# 2. Mount the Container (Enter Password manually this one time)
sudo veracrypt --text --pim=0 --keyfiles="" --protect-hidden=no /home/user/scripts/ctrl_s_master/vaults.hc /mnt/secure_vaults

# 3. Create Internal Directory Structure
cd /mnt/secure_vaults
sudo mkdir -p vaults/json vaults/kdbx 2fa backups

# 4. MOVE the .env file inside (The Phantom Strategy)
# Assuming you created the .env locally to configure it first
sudo mv /home/user/scripts/ctrl_s_master/.env /mnt/secure_vaults/.env

# 5. PERMISSIONS (CRITICAL STEP)
# This ensures the script (running as you/root) can write, 
# and you can read the files later.
sudo chown -R user:user /mnt/secure_vaults

# 6. Unmount
cd ~
sudo veracrypt --text --dismount /home/user/scripts/ctrl_s_master/vaults.hc

---

4. The Automation Schedule
sudo crontab -e
Paste:
0 4 8-14,22-28 * * [ "$(date +\%u)" = 5 ] && /home/user/scripts/ctrl_s_master/run.sh
